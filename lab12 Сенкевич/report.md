# Отчёт по тестированию ПО — Веб-приложение “DeepSeek”

## 1. Введение
**Цель работы:**  
Провести системное тестирование веб-приложения “DeepSeek” для проверки корректности работы функционала, выявления дефектов и оценки соответствия требованиям.  

**Краткое описание программы:**  
DeepSeek — это диалоговый ИИ ассистент на основе большой языковой модели. Приложение позволяет пользователю вести текстовые диалоги, загружать файлы (изображения, PDF, Word, Excel, текстовые документы) для анализа содержимого, просматривать историю чатов, переименовывать и удалять диалоги, а также использовать контекстно-зависимые ответы в рамках одного сеанса.

**Обоснование тестирования:**  
Системное тестирование необходимо для оценки стабильности работы веб-интерфейса с точки зрения конечного пользователя, проверки корректности обработки запросов моделью, целостности загружаемых файлов и удобства навигации по истории чатов.

## 2. Тест-план (Test Plan)

### 2.1 Цели и задачи
- Проверка корректности отправки текстовых сообщений и получения ответов от модели
- Проверка функционала загрузки и анализа файлов различных форматов
- Проверка работы с историей чатов (создание, переименование, удаление)
- Проверка корректного отображения элементов интерфейса
- Проверка обработки длинных диалогов и сохранения контекста
- Выявление ошибок и дефектов
- Проверка соответствия поведения приложения ожиданиям пользователя

### 2.2 Область применения
**Тестируемые модули:**  
- Модуль отправки сообщений и потоковой выдачи ответа
- Модуль загрузки и обработки вложенных файлов
- Модуль управления чатами (боковая панель истории)
- Модуль переименования и удаления диалогов
- Модуль навигации и создания нового чата

**Ограничения:**  
- Тестирование проводится только на десктопной версии в браузере Google Chrome
- Не тестируются мобильные версии, альтернативные браузеры и нативное API
- Не проверяется работа модели в офлайн-режиме

### 2.3 Стратегия тестирования
- Функциональное тестирование всех указанных модулей
- Регрессионное тестирование после обновлений интерфейса
- Тестирование GUI для проверки вёрстки и адаптивности элементов
- Smoke-тестирование при каждом входе в приложение
- Исследовательское тестирование для выявления краевых случаев поведения модели


### 2.4 Критерии начала тестирования
- Приложение доступно по URL и авторизация выполнена успешно
- Стартовая страница загружается без ошибок в консоли браузера
- Реализован базовый функционал отправки сообщений и отображения истории

### 2.5 Критерии завершения тестирования
- Выполнено ≥ 90% тест-кейсов
- Все критические баги зарегистрированы и описаны
- Основные сценарии использования (задать вопрос, загрузить файл, создать новый чат) работают стабильно

### 2.6 Тестовая среда
- **ОС:** Windows 10 Pro (64-bit)  
- **Браузер:** Chrome v112    
- **Дополнительно:** интернет-соединение, набор тестовых файлов для загрузки (PDF, DOCX, PNG, TXT), утилита для снятия скриншотов
